The key thing in those docs is your VMS API supports webhooks (webhookUrl) and SSE. That makes this a perfect fit for the cheapest possible AWS setup.

The architecture that costs ~$0/month
Instead of one long-running process polling a 10-minute translate job, use a webhook chain:


text
User → Telegram → [Lambda A: start job, store chatId] → VMS API (with webhookUrl)
                          ↑                              │ (heavy work, 5–15 min)
                          └── [Lambda B: VMS calls back] ┘ → sends Telegram message
Both Lambda invocations are sub-second. The slow work happens on your VMS API, not on AWS. So AWS never bills you for waiting.

Component	AWS service	Monthly cost
HTTPS endpoint for Telegram + VMS webhooks	Lambda Function URL	$0 (free tier: 1M calls/mo)
Store jobId → chatId mapping (stateless Lambdas)	DynamoDB	$0 (25GB + generous RCU/WCU free tier)
Secret (bot token, VMS key)	SSM Parameter Store (standard)	$0
Total for a personal bot		≈ $0.00/month
You stay in free tier easily unless you do thousands of jobs/day.

Why NOT polling here (important)
Video translate/dub jobs run 5–15+ minutes. If you poll inside Lambda, you hit its 15-minute hard timeout → jobs die. The webhook approach has no such limit — the VMS API does the waiting and just pings you once when done. This is the single biggest reason to go webhook+Lambda over a polling loop.

The simpler alternative: $3.50/mo flat
If you'd rather write straightforward code (polling loop, one Node process, no DynamoDB, no webhook plumbing): AWS Lightsail $3.50 instance (512MB, 1 vCPU). Flat price, no surprises, easy SSH. Just be aware long translate jobs need your VMS API's webhook or you poll from the VM (a VM has no timeout, so polling is fine there).

My recommendation
Webhook + Lambda if you want it as close to free as possible and don't mind slightly more plumbing. Lightsail $3.50 if you want the simplest code and predictable cost.
